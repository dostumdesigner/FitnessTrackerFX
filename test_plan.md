# План тестирования FitnessTrackerFX

## Цели тестирования
- Проверить, что операции с тренировками (создание, редактирование, удаление, просмотр) работают корректно.
- Проверить фильтрацию и поиск тренировок.
- Проверить сохранение и загрузку тренировок в JSON-файл.
- Проверить обновление статистики (количество тренировок, суммарная длительность, средняя длительность).

## Объект тестирования
Приложение **FitnessTrackerFX** — JavaFX-приложение для планирования и учёта тренировок.

Модули:
- вкладка «Тренировки» (таблица, кнопки CRUD, поиск/фильтр),
- вкладка «Статистика»,
- окно добавления/редактирования тренировки,
- сохранение и загрузка данных (JSON).

## Среда тестирования
- ОС: Windows 10 / 11  
- Java: JDK 17  
- JavaFX  
- IDE: IntelliJ IDEA  
- Библиотека: Gson (для JSON)

## Основные сценарии
1. Создать новую тренировку с корректными данными.
2. Попробовать создать тренировку с пустыми обязательными полями (должна сработать валидация).
3. Отредактировать существующую тренировку.
4. Удалить тренировку с подтверждением.
5. Отменить удаление тренировки.
6. Отфильтровать тренировки по типу (например, «Кардио»).
7. Найти тренировку по части названия или комментария.
8. Сохранить список тренировок в JSON-файл.
9. Загрузить тренировки из JSON-файла.
10. Проверить, что статистика обновляется после добавления/удаления/изменения тренировок.
11. Дать новому пользователю добавить и найти тренировку без подсказок.
