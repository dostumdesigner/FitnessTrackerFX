# Отчёт о тестировании FitnessTrackerFX

## Таблица тест-кейсов

| ID   | Тип            | Что проверяем                                | Краткий результат                                        | Статус |
|------|----------------|----------------------------------------------|----------------------------------------------------------|--------|
| TC01 | Функциональный | Добавление новой тренировки                  | Тренировка добавлена и видна в таблице                   | Pass   |
| TC02 | Функциональный | Валидация при пустых обязательных полях     | Показан Alert, тренировка не создалась                   | Pass   |
| TC03 | Функциональный | Редактирование тренировки                    | Данные тренировки обновились в таблице                   | Pass   |
| TC04 | Функциональный | Удаление тренировки с подтверждением        | После «Да» тренировка удалена                           | Pass   |
| TC05 | Функциональный | Отмена удаления                              | После «Нет» тренировка остаётся в списке                 | Pass   |
| TC06 | Функциональный | Фильтрация по типу тренировки                | Показаны только выбранный тип (например, «Кардио»)       | Pass   |
| TC07 | Функциональный | Поиск по части названия/комментария         | Показаны только подходящие тренировки                    | Pass   |
| TC08 | Функциональный | Сохранение тренировок в JSON                 | Создан JSON-файл, в нём все текущие тренировки           | Pass   |
| TC09 | Функциональный | Загрузка тренировок из JSON                  | Таблица заполнилась данными из файла                     | Pass   |
| TC10 | Функциональный | Обновление статистики                        | Цифры меняются после добавления/удаления тренировок      | Pass   |
| TC11 | Пользовательский | Новый пользователь добавляет тренировку   | Большинство смогли добавить тренировку без подсказок     | Pass*  |
| TC12 | Пользовательский | Понимание вкладки «Статистика»            | В целом понятно, но есть идеи по улучшению               | Pass*  |
